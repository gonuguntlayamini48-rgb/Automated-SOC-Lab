# 🛡️ Automated DevSecOps & Incident Response Lab

## 🚀 Overview
This project demonstrates a production-grade security workflow. It combines **Vulnerability Management** (Shift-Left security) with **Automated Incident Response** to reduce the mean time to respond (MTTR) to cyber threats.

---

## 🛠️ Key Components

### 1. Shift-Left Security (GitHub Actions + Trivy)
I implemented a CI/CD pipeline that automatically scans container images for vulnerabilities every time code is pushed.
* **Tool:** Trivy
* **Benefit:** Ensures that critical CVEs never reach the production environment.

### 2. Automated Incident Response (Python)
I developed a custom Python orchestration script (`active_response.py`) that simulates how a SOC (Security Operations Center) would programmatically block attackers.
* **Logic:** The script parses simulated SIEM alerts and triggers firewall "DROP" rules for malicious IPs.
* **Benefit:** Replaces manual intervention with millisecond-speed automation.

---

## 📊 How to View the Results
1. **Security Scans:** Click on the **[Actions]** tab at the top of this repo to see the live vulnerability reports generated by the automated scanner.
2. **Logic Engine:** Open `active_response.py` to see the automated blocking logic.

## 💡 Interview Talking Points (For Hiring Teams)
* **Why this architecture?** It follows the modern "Defense in Depth" strategy.
* **Scalability:** The Python logic can be easily integrated with AWS Lambda or Palo Alto APIs for real-world cloud security.
* **Vulnerability Management:** By using Trivy, I am focusing on high-risk vulnerabilities (CRITICAL/HIGH) to reduce "alert fatigue."
